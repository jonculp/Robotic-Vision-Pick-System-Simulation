Model architecture
▌ Encoder (left, blue)
Each block halves the spatial resolution (320→160→80→40px) while doubling the number of feature channels (64→128→256). This forces the network to extract progressively more abstract features — edges become textures, textures become shapes. Block height in the diagram represents spatial size; the shrinking shape shows compression.
▌ Bottleneck (center, bright blue)
The smallest block — 512 channels at the lowest resolution. This is where the network's full understanding of the scene is encoded. Every pixel's disparity must be derivable from this compact representation. In production networks this is also where global context (scene layout, object boundaries) is captured.
▌ Decoder (right, green)
Mirrors the encoder in reverse — each block upsamples back toward full resolution, reducing channels. Without skip connections, fine detail lost during encoding cannot be recovered. The decoder reconstructs where disparity boundaries are; the encoder learned what caused them.
▌ Skip connections (dashed lines)
Direct shortcuts that copy encoder feature maps across to the matching decoder layer at the same resolution. They preserve fine edge detail (object outlines, surface texture boundaries) that would otherwise blur out during the compression-expansion cycle. This is the key innovation of U-Net over earlier encoder-decoder designs.
▌ Why ~85% accuracy ceiling?
No stereo algorithm — learned or classical — can match pixels it cannot see. Occlusion boundaries (where one object hides another), featureless surfaces (plain walls, shadows), and specular highlights all produce invalid or ambiguous matches. Real RealSense and Azure Kinect cameras hit the same ceiling; they use IR dot patterns to reduce featureless regions, which is exactly what this sim's speckle texture approximates.
▌ What this sim is actually doing
The training here is not real backpropagation. Instead each epoch linearly blends the SAD disparity output toward the ground truth (GT) disparity from the GPU depth pass — approximating how a real network would converge. This lets you see what a trained network's output would look like relative to classical SAD, without running actual matrix multiplication on 42,000 pixels per frame in the browser.

How it works — key concepts
steps 1–2
Stereo vision — Two cameras at a known horizontal offset (baseline). Each sees the same scene from a slightly different angle. Objects closer to the cameras appear shifted further apart between the two images (high disparity); distant objects shift less. An aperiodic dot pattern is stamped into both camera pixel buffers before matching runs, simulating an active stereo IR emitter — the same technique used by Intel RealSense.
steps 1–2
Active stereo IR pattern — A Poisson disk IR dot pattern is projected onto the scene (platform, parts, arm) and stamped into the grayscale buffers used for stereo matching — but never into the RGB camera display. This mirrors real active-stereo cameras (RealSense, Azure Kinect): an IR projector emits a dot pattern invisible to human eyes and RGB cameras, while a dedicated IR camera pair uses it for matching. Poisson disk spacing guarantees every local patch is unique, giving the block matcher an unambiguous winner at each search position.
step 3
Disparity map — For every pixel in the left image, the algorithm searches the right image on the same horizontal scanline to find the best match. The pixel shift (in px) is the disparity value. Shown here with a turbo colormap: blue = far, red = near.
step 3
Depth formula Z = f·B/d — Converts disparity to metric depth. f = focal length in pixels, B = baseline in meters, d = disparity in pixels. A disparity of 82px with f=246px and B=1.70m gives Z ≈ 5.1m.
learned matching
U-Net learned stereo — Production systems replace SAD/SGBM with a trained encoder-decoder network (U-Net architecture). The encoder extracts features at multiple scales; skip connections preserve fine edge detail; the decoder upsamples back to full resolution. Trained on GT disparity supervision — exactly the 42k px/frame ground truth this sim generates. The training simulation below shows convergence: voids fill in, noisy matches sharpen, accuracy asymptotes at ~85% (real network ceiling due to textureless regions and occlusion boundaries).
steps 1–3
SAD stereo matching — Sum of Absolute Differences (SAD) compares pixel blocks between left and right images. An L-R consistency check discards unreliable matches, a median filter removes noise, and color-guided propagation fills gaps in uniform-color regions.
disparity (GT)
Ground truth disparity — A GPU depth pass from stereoL's exact position gives per-pixel true Z. Disparity is then derived as d = f·B/Z — the inverse of the depth formula. Auto-ranged so the small depth difference between parts (~6.1m) and platform (~6.4m) spreads across the full turbo colormap. Three zones: near-black = background; dark steel-blue = arm/rig structure; full colormap = work zone. Compare directly with the computed disparity panel to see exactly where the stereo algorithm loses information.
calibration
Reprojection error — Measures stereo calibration quality: how many pixels a known 3D point projects away from its expected image position. Below 0.5px is good; above 0.5px indicates calibration drift and degrades depth accuracy.
steps 5–6
Inverse kinematics (IK) — Given a target XYZ position for the end effector (suction cup), IK works backward through the arm's joint geometry to compute the joint angles needed to reach it. This sim uses a closed-form analytic solver for a 2-DOF planar arm.
